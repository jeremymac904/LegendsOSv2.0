This file contains NO secret values — only names and exact
instructions. Jeremy is the only person who handles values.

----------------------------------------------------------
KEY 1 (REQUIRED): GOAT_COMMAND_API_KEY
----------------------------------------------------------
What it is:
  The Bearer token the GOAT Architect Custom GPT sends to
  https://legendsos.app/api/goat/*. Every command endpoint
  fails closed (503/401) without it.

Where the value already lives:
  - /Users/JeremyMcDonald/Desktop/LegendsOSv2.0/.env.local
    (line: GOAT_COMMAND_API_KEY=...) — added automatically,
    never committed to git.
  - Netlify site env (set automatically if the MCP update
    succeeded — verify under Site configuration →
    Environment variables → GOAT_COMMAND_API_KEY).

What Jeremy does in GPT Builder:
  1. Open your GOAT Architect GPT → Configure → Actions.
  2. Import schema from URL:
       https://legendsos.app/api/goat/openapi
  3. Authentication → API Key → Auth Type: Bearer.
  4. Paste the value of GOAT_COMMAND_API_KEY from .env.local.
  5. Test the get_health action — it should return
     status "ok" and auth_configured true.

----------------------------------------------------------
KEY 2 (OPTIONAL): GITHUB_TOKEN
----------------------------------------------------------
What it is:
  A GitHub personal access token (classic, `repo` read is
  enough). Unlocks live repo status in get_repo_status and
  higher rate limits for research_github. Everything works
  without it — research just runs at the low anonymous
  rate limit.

Where to add it (only if you want it):
  - Netlify → Site configuration → Environment variables →
    add GITHUB_TOKEN = <your token> → redeploy.
  - Locally: add GITHUB_TOKEN=<your token> to .env.local.

RULES:
  - Never commit values to git.
  - Never paste values into ChatGPT/Claude chats.
  - The prepare_secret_file endpoint generates files like
    this one for any future keys — it rejects real secrets
    by design.
